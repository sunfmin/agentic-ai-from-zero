# 第 1 章 —— 认识终端

_上次校对：2026-05，对齐 Claude Code v2.x_

终端是你 Mac 上的一个窗口，你在里面敲命令、而不是点鼠标。就这样。剩下要学的全是"敲什么"。读完这一章你会掌握五个命令 —— 这五个就够你读完后面所有章节用的了。

## 打开 Terminal.app

按 `Cmd+Space` 打开 Spotlight，输入 **Terminal**，回车。一个带着一些文字的窗口就弹出来了。看上去类似：

```
yourname@Mac ~ %
```

最后这一行叫做 **prompt**（提示符）。后面那个闪烁的方块是你的光标 —— 你敲的东西会出现在那儿。`%` 前面的文字是电脑在告诉你"你是谁"（`yourname`）和"你在哪儿"（`~`，意思是"家"）。

这个窗口接下来要陪你读完大半本书。你可以拖动它的大小，也可以以后换它的配色 —— 不过现在先别动，跟它打个照面就好。

> 🛠 试一试
>
> 通过 Spotlight 打开 Terminal.app。
>
> 你应该看到：一个窗口，prompt 以 `%`（或者老一点配置上是 `$`）结尾，后面有个光标。

## 1. `pwd` —— 我在哪儿？

终端永远知道你"在哪个文件夹里" —— 当前所在的文件夹。命令 `pwd`（print working directory）会告诉你具体是哪一个。

> 🛠 试一试
>
> 输入：`pwd`，回车。
>
> 你应该看到：一条像 `/Users/yourname` 这样的路径。这是你的家目录，从硬盘根目录写下来的完整路径。

`/Users/yourname` 和 `~` 是同一个意思 —— 后者只是写法的简写。

## 2. `ls` —— 这儿有什么？

`ls`（list）列出当前文件夹里所有文件和子文件夹。

> 🛠 试一试
>
> 输入：`ls`，回车。
>
> 你应该看到：一串文件夹名 —— Desktop、Documents、Downloads 之类，可能还有几个。这些就是你在 Finder 左边栏点房子图标看到的那些。

如果 `ls` 什么都没打印，说明你在一个空文件夹里。这也是合理结果。

## 3. `cd` —— 去别的地方

`cd`（change directory）把你"带到"另一个文件夹里。给它一个文件夹名，你就进入了那个文件夹。

> 🛠 试一试
>
> 输入：`cd Desktop`，回车。
>
> 再输入：`pwd`。
>
> 你应该看到：`/Users/yourname/Desktop`。你现在"在" Desktop 里面了。

要回到上一层 —— 从 Desktop 退出来、回到家目录 —— 用两个点：

> 🛠 试一试
>
> 输入：`cd ..`，回车。
>
> 再 `pwd`。
>
> 你应该看到：又变回了 `/Users/yourname`。两个点的意思是"包含当前文件夹的那个文件夹"。

如果你迷路了，光敲 `cd` 不带任何参数会直接把你带回家。`cd ~` 也一样。

## 4. `mkdir` —— 建一个新文件夹

`mkdir`（make directory）创建一个新的空文件夹，名字你说了算。

> 🛠 试一试
>
> 输入：`mkdir my-first-folder`，回车。
>
> 再 `ls`。
>
> 你应该看到：原来的文件夹列表里多了一个 `my-first-folder`。它现在是空的。

文件夹名最好不要带空格，敲起来更省事。本书之后凡是要起名字的地方都用连字符（dash）分单词，原因就是这个。

## 5. `open` —— 回到 Finder 的桥

`open` 把一个文件或文件夹用"双击它"的方式打开 —— 跟你在 Finder 里双击的效果一样。

> 🛠 试一试
>
> 输入：`open .`，回车。
>
> 你应该看到：一个 Finder 窗口弹出来，显示的就是你当前所在的文件夹。那个点（`.`）的意思是"我现在在的文件夹"。

这是你的退路。任何时候终端让你犯怵，敲一句 `open .` 就能立刻回到你熟悉的 Finder 界面。

## 小结

| 命令 | 它做什么 |
|---|---|
| `pwd` | 打印当前所在的文件夹 |
| `ls` | 列出当前文件夹里有什么 |
| `cd <名字>` | 进入一个文件夹 |
| `mkdir <名字>` | 新建一个文件夹 |
| `open <名字>` | 在 Finder 里打开一个文件或文件夹 |

这五个命令就是你读完本书剩下所有章节需要的全部。如果还觉得手生，把每一个 🛠 框再做一遍，直到"你应该看到"那一句对你来说变得理所当然。
