# 第 9 章 —— 三个练习

_上次校对：2026-05，对齐 Claude Code v2.x_

读到这儿，你该从"读者"切换成"做的人"了。三个练习 —— 一个设计师风味、一个 PM 风味、一个通用。每一个都有题面、提示，以及一份跟这一章一起提交的**参考 `SKILL.md`** —— 等你自己动手做完之后再去对一下。

参考答案不是评分标准，只是每个练习的一种合理解法。你写的可能比它更好。

## 练习 1（设计师）—— 设计 brief 审稿器

**题面。** 设计 brief 经常半截就送到你手上。你想要一个 skill：给它一份 brief markdown 文件，它告诉你哪些标准 section 缺了或者含糊：受众、成功标准、约束、交付物。

**触发面。** 你可能会跟同事说的那些话："这份 brief 帮我把把关"、"审一下这份 brief"、"这份 brief 缺什么"。description 要在这三种说法上都触发。

**提示。**

- skill 的 body 应该明确列出那四个必备 section。别让 Claude 自己猜。
- 输出格式很关键。一份只说"看着还行"的"审稿"没用；一份对每个 section 都标上"齐全 / 含糊 / 缺失"、对"含糊"和"缺失"给出具体建议的，才是可执行的。
- 你得想清楚"含糊"是什么。一个把所有东西都判定为含糊的 skill，跟一个对任何东西都不判含糊的 skill，一样糟。要具体。

**做完之后。** 对一下 `tutorial/skills/exercises/design-brief-reviewer/SKILL.md`。

## 练习 2（PM）—— PRD note 起草器

**题面。** 你从会议上下来，Notion 页面里一堆乱糟糟的 bullet、Slack 引文、半截决定。你想要一个 skill，把它整理成一份结构化的 PRD note：问题、提议方案、待解决问题、已决定的下一步、风险。

**触发面。** "把这个变成 PRD"、"从这些起草一个 PRD note"、"抽出 PRD"，以及任何相近变体。还有：粘一段粗糙内容 + 要求"把这个结构化一下"。

**提示。**

- 这就是你在第 8 章用 `/skill-creator` 建过的那个 skill。如果你存了下来，大部分活儿已经做完了。这个练习的目的是**打磨**它：哪些触发说法接不住？面对真实乱糟糟的输入时，输出长得怎么样？
- 最难的不是 section 列表 —— 是决定**输入里没有某个 section 内容时 skill 怎么办**。硬编码"（输入里没有）"，还是让 Claude 推测？参考答案选了显式那条路。

**做完之后。** 对一下 `tutorial/skills/exercises/prd-note-drafter/SKILL.md`。

## 练习 3（通用）—— 会议 action item 提取器

**题面。** 你做粗糙的会议纪要 —— 段落、bullet、提到的人名。你想要一个 skill，把 action item 抽出来，每一条带 owner 和 due date（或者标记为缺失）。

**触发面。** "抽 action"、"拉出 action item"、"我们都同意干啥了"、"列一下这个会要做的 todo"。还有：粘会议内容 + "谁负责哪一项？"

**提示。**

- 输出格式是承重决策。"编号列表 + owner-action-date"容易扫读、容易验证；一段散文式的描述不行。
- 这个 skill 必须**拒绝凭空指派 owner**。如果纪要说"得测一下迁移"但没指名道姓，那 action 的 owner 是"未指派" —— 不是"团队"、不是"大概是工程组"。把含糊的地方明确标出来，让人来定。
- 最难的 case：隐式承诺。"我们被 auth 那个 bug 卡了" —— 这算 action 吗？谁的？想清楚、写明白。

**做完之后。** 对一下 `tutorial/skills/exercises/meeting-action-extractor/SKILL.md`。

## 每个练习怎么做

对每一个：

1. 在 `~/.claude/skills/<skill-name>/` 下建 skill 文件夹。
2. 手写 `SKILL.md`，**或者**用 `/skill-creator` 让它生一份再调。
3. 用感觉真实的输入测一下 —— 没真实材料就自己编一段，但要乱、要有真实感。
4. 如果 skill 不能稳定触发，按第 8 章的排错清单逐项过。
5. 等 skill 按你想要的样子工作之后，**再**打开参考答案读。注意差异。判断参考写得更好、更差，还是只是不一样。

## 小结

- 三个练习：设计师、PM、通用。
- 参考答案存在但不是评分标准 —— 你写的可能更好。
- 练习 1 和 3 的意义是：让你结束第 8 章时，手上有"**你做的** skill"，而不只是"书替你做的 skill"。
