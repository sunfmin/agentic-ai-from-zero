# 第 3 章 —— 你的第一次对话

_上次校对：2026-05，对齐 Claude Code v2.x_

你已经登录了。接下来这一步最简单、也最关键：让 Claude 在你电脑上做三件事。每一件做完之后，我们再回头给"刚刚发生了什么"起个名字。

这一章的形态就这样。🛠 框先走，概念后到。先做框，先看到事情发生，再读那段给它命名的文字。这样比反过来好用。

## 准备 —— 一个干净的沙盒文件夹

先别在真实项目里跑 `claude`。给这一章建一个空文件夹，在里面玩。

> 🛠 试一试
>
> 输入：`cd ~ && mkdir -p first-session && cd first-session`
>
> 然后：`pwd`
>
> 你应该看到：`/Users/yourname/first-session`。

现在启动 Claude Code：

> 🛠 试一试
>
> 输入：`claude`
>
> 你应该看到：Claude Code 的几行欢迎信息，然后是等你输入的 prompt。光标落在一行以 `>` 开头的位置上。

进来了。你打的任何东西现在都发给 Claude。

## 互动 1 —— 让 Claude 读一个文件

先给 Claude 一点东西读。我们手上的终端还能继续用 —— 不过 Claude 也能帮我们干这一步。看着。

> 🛠 试一试
>
> 在 Claude 的 prompt 上输入：
>
> > 在这里建一个叫 `notes.md` 的文件，里面写三行关于你今天上午的随手记。
>
> 你应该看到：Claude 宣布要写文件。它可能会让你确认写入操作 —— 答应。然后它确认 `notes.md` 写好了。如果你另开一个 Terminal 窗口在 `first-session` 里 `ls` 一下，能看到 `notes.md`。

现在让它把刚才写的读回来：

> 🛠 试一试
>
> 在 Claude 的 prompt 上输入：
>
> > 读一下 `notes.md`，把里面的内容告诉我。
>
> 你应该看到：Claude 把刚才那个文件读出来，把三行内容引述给你。

刚才发生了什么：Claude 读写文件，是**在你当前的文件夹里**做的 —— 你从哪儿启动 `claude` 它就在哪儿。那个文件夹有个名字，叫 **工作目录**。Claude 做的所有事 —— 读、写、跑 —— 都是相对这个目录的。从 `~/Desktop` 启动 `claude`，它能读 Desktop 的文件。从 `~/first-session` 启动，它就只能读那一个文件夹里的东西。

这是个特性，不是限制。Claude 不会"不小心"溜进你电脑的其他角落乱读。

## 互动 2 —— 让 Claude 写一个新文件

互动 1 里你已经让它写过一个文件了，不过那次内容是 Claude 自己编的。现在试着给具体要求。

> 🛠 试一试
>
> 在 Claude 的 prompt 上输入：
>
> > 建一个叫 `todo.md` 的文件，里面写三条我今天可能想做的事。要现实、要短。
>
> 你应该看到：Claude 把内容草拟出来，问能不能写文件（或者写完之后向你确认），然后报告成功。从另一个 Terminal `ls` 一下，`todo.md` 跟 `notes.md` 并排出现。

也可以用对话方式重新打开并编辑文件：

> 🛠 试一试
>
> 在 Claude 的 prompt 上输入：
>
> > 打开 `todo.md`，加一条第四项："读 agentic-ai 教程第 4 章"。
>
> 你应该看到：Claude 把文件读进来、改、写回去。在任何编辑器里打开 `todo.md`（或者另一个 terminal 里 `cat todo.md`），新加的那一条就在那儿。

刚才发生了什么：Claude 要动你的文件系统时，它会先问你。可能是每个动作问一次，可能是每类动作问一次 —— 这个等以后可以调。这些问你"行不行"的提示叫 **权限提示**。你可以批准单次、批准本 session 内的某一类、或者拒绝。默认偏保守，这阶段挺合适。

## 互动 3 —— 让 Claude 跑一条命令

终端里有各种命令。Claude 也能跑 —— 走同一套权限系统。

> 🛠 试一试
>
> 在 Claude 的 prompt 上输入：
>
> > 在这里跑一下 `ls -la`，把看到的东西总结一下。
>
> 你应该看到：Claude 宣布要跑 `ls -la`，问你权限，跑完之后把结果读回来 —— `notes.md`、`todo.md`，加上隐藏的 `.` 和 `..` 项。

刚才发生了什么：这整段交流 —— 你的 prompt、Claude 的动作、中间所有的权限提示 —— 加起来是一次 **会话**（session）。会话从你跑 `claude` 那一刻开始，到你按 `Ctrl+C` 或者输入 `/exit` 结束。Claude 在会话内记住的一切只在这次会话内有效；退出之后大部分就忘了（不是全部 —— 第 4 章讲的 `CLAUDE.md` 正是那一份**跨会话保留**的东西）。

## 退出 session

> 🛠 试一试
>
> 在 Claude 的 prompt 上输入：`/exit`
>
> 或者按两下 `Ctrl+C`。
>
> 你应该看到：Claude 打一段告别，你回到普通 shell prompt（`%` 或 `$`）。你刚建的文件还在硬盘上。

## 小结 —— 给三个概念命名

- **会话（session）** —— 一次完整对话，从 `claude` 启动到 `/exit`。记忆只在 session 内有效。
- **工作目录** —— 你启动 `claude` 时所在的文件夹。Claude 的读写、跑命令都只能相对这个目录发生。
- **权限提示** —— Claude 要动文件系统或者跑 shell 命令之前，会先问你行不行。你可以单次批准、按类批准、或者拒绝。

这三件事你这一章里都用了，但事先没人告诉你它们叫什么名字。这个顺序是对的：先体验，再命名。
